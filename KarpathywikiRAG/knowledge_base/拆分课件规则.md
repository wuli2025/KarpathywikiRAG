## 一、通用基礎

### 1.1 三層鐵律（強制）

| 層 | 路徑 | 規則 |
|---|------|------|
| 原始層 | `raw/整理课件/` | **絕對只讀，禁止任何修改** |
| 分析層 | 分析記錄文件 | 過程日誌，追加不修改歷史 |
| 產出層 | `<根目錄>/4.资料/拆分课件/` | 乾淨的主題文件，等待用戶確認後再 Ingest |

### 1.2 文件命名規範

```
[層級代碼]_[內容類別]_[主題]_[版本號].md
```

**示例**：
- `L3_功法_中脈動功五式_V1.md`
- `L4_知見_法身開通條件_V1.md`
- `L6_修炼_坐禅行禅_V1.md`
- `L6-第11届-交易_均線賺賠比_V1.md`（L6 屆次專屬格式）

**多版本課程命名警告**：同一課程有 V1、V2…多個版本時，每版文件名必須以**本版實際內容**為準，不能從前版複製修改。

### 1.3 內容分類體系

**課程層級**：L1 公開課 / L2 任督課 / L3 中脈課 / L4 法身課 / L5 弟子班 / L6 經天會 / L7 共修 / 跨層級

**內容類別（目錄名）**：
- `功法/`：有具體動作操作說明的練功方法
- `知見/`：理論講述、原理說明、故事案例、世界觀
- `儀軌/`：咒語、祈請文、宣告、迴向收功等儀式性內容
- `升班/`：升班邀約、下一步路徑、課程介紹

L6 專用目錄：`L6_修炼/` / `L6_商业/` / `L6_架构/`

### 1.4 分析記錄

每次拆分必須建一份分析記錄（追加式日誌），存放在產出目錄旁：

```
分析记录_YYYYMMDD-HHMM.md
```

必含四節：方案決策 → 原始骨架 → 拆分映射方案 → 執行結果。

## 二、通用拆分原則（方案 C）

### 2.1 核心邏輯

天脈課程拆分統一採用**方案 C：按獨立大主題拆**。

| 方案 | 文件數 | 優點 | 致命缺點 |
|------|-------|------|---------|
| A：按內容類型拆 | 8–10 | 類型邊界清楚 | 單文件雜燴、過大 |
| B：按每堂拆 | 10–11 | 課堂脈絡保留 | 同主題分散多堂 |
| **C：按獨立大主題拆** | **14–20** | **主題獨立 + 跨堂合併 + 下游可直接命中** | 課堂時間脈絡丟失（靠 INDEX 雙向映射追溯）|

**執行順序**：
1. 先列出所有獨立主題（不看堂次，只看「什麼功法/什麼知見」）
2. 為每個主題找出全部課堂中的相關段落（跨堂蒐集）
3. 匯總成一個文件，來源行號用 ` + ` 連接多段

### 2.2 跨堂合併判斷規則

| 情形 | 操作 |
|------|------|
| 同一功法在多堂出現 | 合併到一個文件 |
| 同一知見主題跨多堂講述 | 合併到一個文件 |
| 同類儀軌內容分散在各堂 | 匯總到儀軌專屬文件 |
| 同一堂內包含完全不同的主題 | 拆開放不同文件 |

**功法文件不受最小篇幅限制**：凡含具體動作操作說明的段落，無論行數多少（哪怕只有 30 行），單獨建功法文件——功法（「怎麼做」）與知見（「為什麼」）的檢索場景不同。

### 2.3 大塊內容無分隔標題的處理

**觸發**：提取後單文件超過 2000 行。此時不是結論，只是提示需要抽樣檢查。

**兩步檢查**：

```bash
# 第一步：查內部標題
grep -n "^#" 文件名.md
# 有多個標題 → 內部已有結構，通常不需處理
# 只有開頭一個標題 → 進行第二步

# 第二步：三點抽樣（各讀 20–30 行）
# 文件 1/4 處、1/2 處、3/4 處各抽一段
```

**判斷**：三處主題連貫 → 放行；發現明顯主題跳躍 → 登記到 `1.需要細拆文章.md` 待後續處理。

**判斷口訣**：同一個人讀這三段，能不能自然地說「這是同一篇文章」？能 → 放行；不能（領域都變了）→ 登記。

### 2.4 跨屆重複內容的去重標記

**觸發**（滿足其一）：標題完全一致；核心關鍵詞相同僅措辭差異；行數差 ≤ 10%。

**操作原則：先提取，後登記，不跳過**。
1. 正常提取文件（不因疑似重複而跳過）
2. INDEX.md 加標記：`⚠️ 待去重：第N屆同名文件`
3. 登記到 `2.待去重内容.md`

何時不需登記：行數差 > 20%（可能是同名不同文章）。

## 三、路徑 A：對話記錄型

**適用**：含 `### 👤 Human / ### 🤖 Claude` 標記的人機對話記錄。

### 3.1 流程

```
Step A 基礎掃描 → Step B 分段深讀（★★★/★★/★評級）→ 用戶審核 → Step C 7維度標注 → Step D 重疊分析 → 執行提取
```

### 3.2 五種段落類型

| 類型 | 說明 | 處理 |
|------|------|------|
| 廢話段 | 寒暄、確認、「好的我來做」| 跳過 |
| 過程段 | AI 迭代中間版本、被否決方案 | 跳過 |
| 指令段 | 高老師給 AI 的規則和指令 | 提取為「課程規則」|
| 素材段 | 粘貼的原始素材、故事、見證 | 重點提取 |
| 產出段 | AI 生成的最終成品 | 提取 |

**拆分單位是主題，不是消息**。一條消息可能包含多個主題，必須以主題為拆分單位。

### 3.3 提取價值評級

- ★★★：高價值，必須提取（獨立知識塊、完整規則、核心素材）
- ★★：中價值，建議提取
- ★：低價值，可選提取
- 跳過：無提取價值

### 3.4 子代理分工（大文件 >3000 行）

按行號段並行：子代理 1 前 1/3、子代理 2 中 1/3、子代理 3 後 1/3，各輸出內容地圖，主代理合併後執行重疊分析。

## 四、路徑 B：結構化課程型

**適用**：無對話標記、已整理成稿、有「成稿感」的課件。

### 4.1 三個子類型

| 子類型 | 代表課程 | 快速識別 | 邊界來源 |
|--------|---------|---------|---------|
| **有多級標題** | L3 中脈課 | `grep -c "^#" 文件.md` > 10 | 標題行號 |
| **有分隔符** | L4 法身課 | `grep -c "^===" 文件.md` > 0 | 分隔符行號 |
| **純散文逐字稿** | L1 公開課 | 以上兩者均為 0 | 僅靠語義轉折 |

純散文子類型在 Step 1 和 Step 2 之間必須先跑**語義地圖子步驟**：投給 sub-agent 按語義轉折輸出「行號範圍 → 主題名稱 → 類別」表格，確認邊界後再提取。

### 4.2 五步核心流程

#### Step 1：基礎判定 + 全文標題掃描

```bash
wc -l 文件.md                        # 總行數
grep -nE "^#{1,3} " 文件.md          # 提取所有標題與行號
grep -cE "^### 👤 Human" 文件.md     # 確認類型
```

#### Step 2：提出三方案 + 用戶決策（默認推薦方案 C）

列出 A/B/C 三方案對比表，等用戶確認後繼續。

#### Step 3：寫分析記錄

四節：方案決策 → 完整骨架 → 拆分映射方案（含每個合併的理由）→ 執行結果（執行後追加）。

#### Step 4：子代理並行拆分

**按主題分類分工，不按行號段分**。

**Launch 前必做：關鍵邊界驗證**：

```bash
# 驗證計劃行號的實際內容
# 優先驗證：跨堂合併的每個切割點、相鄰段落邊界、最複雜堂次的各段邊界
```

常見陷阱：計劃末行 N 實際是 `---` 或下一堂標題，內容到 N-1 結束，幾乎必現。

**子代理 Prompt 強制規則**（每條都必須寫進 prompt）：

```
1. 保留繁體原文：一字不改、不刪減、不簡化
2. 保留原標題樣式：原文 ## **xxx** 加粗標題原樣保留
3. 節名一律用完整原文標題（含副標題）
4. 不寫 AI 解讀：除文件頭「拆分說明」外，不添加任何 AI 總結
5. YAML 7 字段齊全：課程層級、內容分類、原文件、來源行號、原始堂次、繁簡版本、拆分日期
6. 來源行號多段用 + 分隔（空格加號空格），不用逗號
7. 元數據、〔原始来源〕標記全部使用簡體；原文標題、節名保留繁體
8. 多段合併文件用 ## 〔原始来源 N〕<完整節名>（行 X-Y）作為大節分隔
9. 有跳段就加〔原始来源〕標記，無論是否同一堂；單堂且連續才不加
10. 文件頭加「拆分說明」引用塊（2-3 句話說明包含哪些原始內容、為什麼合併）
11. 嚴格遵守行號上下界，相鄰節歸不同主題文件時絕不越界
12. 功法文件不受最小篇幅限制：凡含具體動作說明的段落，單獨建功法文件
```

#### Step 5：寫 INDEX + 追加分析記錄第 4 次操作

INDEX 是**給讀者用的當下視圖**，必須獨立於分析記錄。

**INDEX 三節標準結構**：

```
一、主題文件清單（按大分類）
   每個文件含：文件名帶鏈接 + 100 字摘要 + 來源行 + 大小

二、原大綱 → 主題文件 映射表
   按原文件章節順序，每節標出歸屬哪個主題文件

三、主題文件 → 原大綱 反查表
   每個主題文件標出來源章節
```

節名一律用完整原文標題（含副標題），不可簡化，確保 Ctrl+F 可搜索命中。

### 4.3 目錄存放約定

```
<課程名>.md                    ← 原文件，永遠只讀
<根目錄>/4.资料/拆分课件/<課程名>/
    ├── INDEX.md
    ├── 分析记录_YYYYMMDD-HHMM.md
    ├── L3_功法_*.md
    ├── L3_知見_*.md
    └── ...
```

### 4.4 交付前必做的 8 項檢查

```bash
DIR="<目標目錄>"

# 1. 文件總數對得上
find "$DIR" -name "*.md" | wc -l

# 2. 全部文件 YAML frontmatter 起始
for f in "$DIR"/**/*.md; do
  [ "$(head -1 $f)" = "---" ] || echo "❌ $f"
done

# 3. 無繁體元數據殘留
grep -rH "〔原始來源" "$DIR"          # 應為空

# 4. 來源行號無逗號（應用 + 分隔）
grep -rh "^来源行号:" "$DIR" | grep "," # 應為空

# 5. INDEX 中文節名無英文冒號
grep -nE "[一-龥]:[一-龥]" "$DIR/INDEX.md"  # 應為空

# 6. INDEX 包含全部內容文件
for f in "$DIR"/**/*.md; do
  fn=$(basename "$f")
  grep -q "$fn" "$DIR/INDEX.md" || echo "❌ MISSING: $fn"
done

# 7. title 與 H1 一致（抽查）
# 8. 改寫狀態分布（合法值：原文引用 / 待改写为高老师语气 / 部分待改写）
grep -rh "^改写状态:" "$DIR" | sort | uniq -c | sort -rn
```

### 4.5 常見踩坑速查

| 坑 | 現象 | 避免方法 |
|----|------|---------|
| 計劃行號末尾差 1 行 | 末行是 `---` 或下一堂標題 | Launch 前用 Read 驗證末行 |
| 無內容堂次沒排除 | 子代理處理空殼堂次 | 掃描大綱時抽讀正文第一段，流程佔位詞（放音樂/迴向）→ 顯式排除 |
| META/結構索引未排除 | 元數據段寫入主題文件 | Prompt 明說「## META 和 ## 結構索引 請跳過」|
| 節名簡化 | INDEX 搜索失效 | 節名必須完整，含副標題 |
| 簡繁體混用 | 3個子代理格式不統一 | Prompt 明確：元數據用簡體，原文保繁體 |
| 英文冒號混入中文 | `三界六道:看清世界` | 完成後跑 `grep -nE "[一-龥]:[一-龥]" INDEX.md` |
| 多段合併文件越界 | 相鄰行號但歸不同主題文件 | Prompt 明確每段精確行號上下界 |
| 同一課堂多主題拆分 | 子代理「自由發揮」邊界 | 不讓子代理決定邊界，主代理精確到行號 |

**跨屆版本保留策略**：

| 行數差 | 操作 |
|--------|------|

每個 `##` 視為一篇獨立文章，逐一評估體量和是否提取。古典文獻類可批量標注「待提取」，不阻塞主流程。

### 5.3 L7 共修課

## 六、執行後確認流程

拆分完成後：

1. 所有產出文件放入 `<根目錄>/4.资料/拆分课件/<課程名>/`
2. 告訴用戶：
   - 產出目錄路徑
   - 文件清單（文件數 + 各文件名 + 行數）
   - 分析記錄路徑
   - 已知問題或待確認項（如去重候選、需細拆文件）
3. **等待用戶確認無誤後**，再詢問是否執行 Ingest 進入知識庫

不得在用戶確認前自動執行 Ingest。
